# Формат отчёта "Умная диагностика"

## 1. Общая структура

Отчёт состоит из двух частей:
- **Текстовый отчёт** (Markdown, `reports/summaries/` и `reports/recommendations/`)
- **Дашборд** (HTML, `reports/dashboards/`)

---

## 2. Текстовый отчёт

### Структура документа

```markdown
# Диагностика: <имя_файла>
> Дата анализа: YYYY-MM-DD
> Источник: <путь к файлу или описание данных>

## 1. Саммари

| Метрика | Значение |
|---|---|
| Строк | N |
| Столбцов | N |
| Период | YYYY-MM-DD – YYYY-MM-DD |
| Пропуски (всего) | N (X%) |
| Дубликаты | N (X%) |

**Краткий вывод:** 1–2 предложения об общем состоянии данных.

## 2. Основные изменения

*Сравнение с предыдущим периодом / контрольной группой / эталоном.*

| Показатель | Было | Стало | Дельта | Δ% |
|---|---|---|---|---|
| <метрика 1> | 100 | 120 | +20 | +20% |
| <метрика 2> | 50 | 45 | -5 | -10% |

**Ключевые изменения:**
- Метрика X выросла на N% — возможная причина: ...
- Метрика Y снизилась на N% — возможная причина: ...

## 3. Проблемы

| № | Тип | Описание | Строки | Серьёзность |
|---|---|---|---|---|
| 1 | Пропуски | Столбец "email" пуст в X строках | 12–15, 23 | Средняя |
| 2 | Выброс | Зарплата 1 000 000 руб. | 42 | Высокая |
| 3 | Дубль | Полный дубликат строки 7 | 18 | Низкая |
| 4 | Аномалия | Резкий спад метрики KPI | авг-2025 | Высокая |

**Топ-3 проблемы для первоочередного решения:**
1. ...
2. ...
3. ...

## 4. Выводы и рекомендации

### Выводы
- ...
- ...

### Рекомендации
| Приоритет | Действие | Ожидаемый эффект |
|---|---|---|
| 🔴 Высокий | ... | ... |
| 🟡 Средний | ... | ... |
| 🟢 Низкий | ... | ... |
```

### Пример заполнения

```markdown
# Диагностика: sales-2026-Q1.csv
> Дата анализа: 2026-06-10
> Источник: CRM-выгрузка за 1 кв. 2026

## 1. Саммари

| Метрика | Значение |
|---|---|
| Строк | 1 245 |
| Столбцов | 12 |
| Период | 2026-01-01 – 2026-03-31 |
| Пропуски (всего) | 47 (3.8%) |
| Дубликаты | 3 (0.2%) |

Данные в целом качественные, основная проблема — пропуски в контактных данных.

## 2. Основные изменения

| Показатель | Q4 2025 | Q1 2026 | Дельта | Δ% |
|---|---|---|---|---|
| Продажи, шт | 2 340 | 2 890 | +550 | +23.5% |
| Средний чек, руб | 4 500 | 4 120 | -380 | -8.4% |
| Конверсия, % | 3.2% | 3.8% | +0.6 п.п. | +18.8% |

Продажи выросли за счёт увеличения трафика, но средний чек снизился.

## 3. Проблемы

| № | Тип | Описание | Строки | Серьёзность |
|---|---|---|---|---|
| 1 | Пропуски | Отсутствует телефон у 23 лидов | 45, 67, 89, ... | Средняя |
| 2 | Выброс | Сумма чека 950 000 руб. | 1024 | Высокая |
| 3 | Аномалия | Просадка конверсии 15–17 марта | 220–280 | Высокая |

**Топ-3:**
1. Разобраться с аномалией конверсии 15–17 марта
2. Проверить корректность выброса по чеку 1024
3. Дозаполнить контактные данные лидов

## 4. Выводы

- Q1 2026 показал уверенный рост продаж (+23.5% к Q4 2025)
- Средний чек снизился — вероятно, из-за сдвига структуры продаж в сторону дешёвых товаров
- Конверсия выросла, но есть аномальная просадка в середине марта (требует отдельного анализа)

## Рекомендации

| Приоритет | Действие | Ожидаемый эффект |
|---|---|---|
| 🔴 Высокий | Проанализировать причины просадки 15–17 марта | Восстановление конверсии |
| 🟡 Средний | Настроить обязательную валидацию контактных данных | Снижение пропусков до <1% |
| 🟢 Низкий | Проверить выброс по чеку 1024 | Корректность отчётности |
```

### Технические требования

- **Фреймворк:** чистый HTML + CSS + JS (Chart.js или Plotly)
- **Адаптивность:** да/нет (на усмотрение)
- **Автономность:** один файл, все зависимости подключаются через CDN
- **Цветовая схема:** светлая тема, акцент — синий (#2563eb)
- **Экспорт:** кнопка "Сохранить как PDF" (опционально)

### Состав дашборда

| Блок | Содержание | Тип |
|---|---|---|
| Шапка | Название файла, дата анализа, кнопка экспорта | — |
| KPI-панель | Строки, столбцы, пропуски (%), дубликаты, изменений (%) | Числа |
| График изменений | Динамика главной метрики по времени | Line chart |
| Распределение | Гистограмма / boxplot выбранной метрики | Bar/histogram |
| Таблица проблем | Список проблем с серьёзностью | Table + color |
| Рекомендации | Действия по приоритетам | Список |

## 4. Варианты без сравнения (первичный анализ)

Если нет данных для сравнения (первая загрузка), раздел "Изменения" заменяется на **"Текущее состояние"**:

```markdown
## 2. Текущее состояние

| Показатель | Значение | Комментарий |
|---|---|---|
| Среднее X | 45.2 | В пределах нормы |
| Медиана Y | 12.0 | ... |
| Std X | 8.7 | Высокая вариативность |
```

Дашборд в этом случае показывает только текущие распределения без динамики.

---

## 5. Формат имени файла отчёта

```
<дата-анализа>_<имя-исходного-файла>_<тип>.md
```

Пример: `2026-06-10_sales-2026-Q1_summary.md`

Для дашборда: `2026-06-10_sales-2026-Q1_dashboard.html`
